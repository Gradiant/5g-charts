** 5G AMF Service **

Internal Access:

For services running at the same k8s namespace, AMF IP can be resolved at `{{ include "common.names.fullname" . }}-amf`.

You can also get the AMF clusterIP with:

```
kubectl get svc -n {{ .Release.Namespace }} {{ include "common.names.fullname" . }}-amf -o jsonpath='{.spec.clusterIP}')
```

{{- if .Values.amf.externalService.enabled }}

Gradiant's lab5g DNS can resolve the External IP of AMF at: 

- externalName: {{ include "common.names.fullname" . }}-mme-ext.{{ .Release.Namespace }}.ext.lab5g.gradiant.org

You can also get the external IP with: 

```
kubectl get svc -n {{ .Release.Namespace }} {{ include "common.names.fullname" . }}-amf-ext -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
```

{{- else }}

External IP Access to this AMF service was not enabled (check amf.externalService.enabled).
